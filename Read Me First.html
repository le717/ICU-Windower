Introduction

This is the Readme for the ICU (LEGO Island Configuration Utility), a Python 3 application to to a provide a collection of tools for the 1997 Mindscape PC game LEGO Island.

ICU Windower

Usage

You have downloaded the ICU Windower component of ICU. This small tool, rewritten from the original Inno Setup version,
automatically enables Windowed Mode and restores the Full Screen Mode, the former of which is a semi-hidden feature of LEGO Island.

Simply click Launch ICU Windower.bat to launch ICU Windower. This release supports both 32- and 64-bit Windows, so there is no need to choose the correct one.
You will need Administrator rights to run ICU Windower, as they are needed to write to the HKEY_LOCAL_MACHINE section of the Registry.

This release automatically sets the Ramp Emulation Game Graphics setting and all other required settings so so Windowed Mode will activate.
It also asks you if you would like to use Ramp Emulation or MMX Emulation when restoring Full Screen Mode, as well as other Game Graphics settings.

From version 2.0 onward, you are able to launch LEGO Island after you activate either Windowed or Full Screen mode.

As of version 2.0.1, you are able to manually select a LEGO Island installation if the auto-detection or game launching fails. However, if you do need to use this feature,
you will have to select the installation every time, as it is not stored between launches. This annoying limitation is planned to be fixed in a future release.

Do note that Windowed Mode is inherently buggy, and it will not activate all the time, or maybe not at all, simply because a method to always activate it is still being developed.
If you experience this, please follow the same steps you would to report a bug.

Support

To request a feature, submit a code patch, or download a new release of ICU, please go to https://github.com/le717/ICU To submit a bug report, please go to https://github.com/le717/ICU/issues, state the bug, and attach YourScoreCube.log, located at *Location of ICU Windower*\Logs. Attaching this file will ensure the bug will be found a lot easier, and a bug fix will be released much faster.
